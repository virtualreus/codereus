# codereus
## Платформа для подготовки к ЕГЭ по информатике.
В современном мире онлайн-образование становится все более востребованным и актуальным среди как крупных образовательных организаций, так и частных репетиторов и онлайн-школ. В связи с массовым переходом на дистанционное обучение, требования к качеству и возможностям образовательных платформ значительно выросли, а некоторые школы и преподаватели и вовсе отказываются от очного обучения в пользу дистанционного.
С 2021 года в России единый государственный экзамен по информатике и информационно-коммуникационным технологиям (далее - ИКТ) стал проводиться в компьютерной форме (далее - КЕГЭ), что полностью исключило все бумажные и рукописные материалы из процесса подготовки к экзамену и его сдачи. Из-за этого многим преподавателям пришлось в значительной степени менять подход к обучению школьников в области ИКТ, и появилась надобность  в создании собственной платформы, где ученики смогут готовиться и решать задачи, а преподаватели – выдавать задания и следить за прогрессом. Если с данным вызовом крупные онлайн-школы справились, то это же массово вызвало трудности среди частных репетиторов, поскольку становление КЕГЭ по ИКТ подразумевает постоянное взаимодействие с компьютером, а пользоваться одним или несколькими общедоступными онлайн-ресурсами, не имея при этом полного доступа к ресурсу и статистике, может оказаться неудобным и нецелесообразным.
Именно поэтому мною и был разработан данный ресурс


### Для создания сайта использован следующий набор языков программирования и инструментов:
•	HTML – основной язык гипертекстовой разметки для создания веб-страниц. Используется для структурирования и организации контента, создавая основу страницы, и определяет заголовки, формы, абзацы, поля и другие элементы.

•	CSS – используется для  стилизации веб-страниц, определяя их внешний вид. С его помощью можно изменять цвета, шрифты, отступы, выравнивание и делать простые анимации.

•	JavaScript (с использованием jQuery) – язык программирования, который используется для создания интерактива между пользователем и веб-страницей, таких как динамическое обновление контента и валидация форм. jQuery – библиотека для JavaScript, упрощающая работу с DOM (Document Object Model), и AJAX-запросами.

•	PHP (с использованием библиотеки RedBeanPHP) – серверный язык программирования, используемый для создания динамических полноценных веб-ресурсов и обработки данных на сервере. Позволяет взаимодействовать с базами данных, управлять пользовательскими сессиями, обрабатывать формы и возвращать (генерировать) HTML-код. RedBeanPHP – библиотека, которая упрощает работу с базами данных методом автоматического создания таблиц и взаимодействия с ними без необходимости написания сложных SQL-запросов, тем самым минимизируя возможность допущения ошибок в написании и полностью исключает SQL-инъекции.

•	phpMyAdmin – веб-интерфейс для управления базой данных MySQL. Предоставляет возможность создания, удаления и редактирования баз данных, таблиц и записей, выполнять SQL-запросы и управлять привилегиями пользователей.

•	Visual Studio Code (VS Code) – текстовой редактор, используемый для написания кода на различных языках программирования. Поддерживает расширения, такие как авто дополнение кода и автоматическое распознавание синтаксических ошибок. Будет использоваться как основной инструмент для написания, редактирования и тестирования кода.

## Теоретическая часть
Далее будут описаны ожидаемые функции данного веб-ресурса, указан теоретический смысл и методы дальнейшей практической реализации. В каждом из следующих пунктов будет рассмотрен функционал как со стороны ученика, так и со стороны преподавателя.
### 1.1. Регистрация и аутентификация
Аккаунт для преподавателя (администратора) создается вручную при помощи phpMyAdmin, ему выдается специальная роль, которая предоставляет доступ к админ-панели и прочим функциям. Регистрация персонального аккаунта для ученика будет производиться через административную панель, для этого будут указаны следующие данные: имя и фамилия ученика, дата онлайн-занятия с ним через любую удобную платформу (при наличии), логин, пароль (придумываются лично преподавателем).
После указания всех необходимых данных создается запись в базе данных, содержащая данную информацию.
Авторизация пользователя проходит на специальной странице, где вводится логин и пароль, функция самостоятельной регистрации не предусмотрена. При вводе корректных данных происходит авторизация и переадесация на страницу с личным кабинетом, в противном случае выводится сообщение о соответствующей ошибке.
### 1.2. Личный кабинет
На странице личного кабинета будет представлен список из всех выданных вариантов и сборников задач и показан их статус (решено, решено частично, не начат), так же будет показано имя, дата и время занятия, а в боковом меню будут ссылки на главную страницу личного кабинета, страницу с заданиями и кнопкой выхода из аккаунта.
### 1.3 Задания
На момент 2024 года в типовом варианте КЕГЭ по информатике представлено 27 различных типов задач, в каждом из которых может быть бесконечное количество вариаций условий, или же они могут быть совершенно разного уровня сложности. В базе данных любое задание содержит следующие поля: номер задания (1-27), уникальный id каждого вида задания, правильный ответ, файл (при надобности: некоторые задания из КЕГЭ нацелены на взаимодействие и анализ различных текстовых или табличных файлов).
Каждое задание вносится вручную через административную панель, доступ к которой имеют только преподаватель. Для внесения задания потребуются следующие данные: номер задания (1-27), фотография условия (вносится как файл через специальное поле), при надобности – файлы для самого задания, правильный ответ, сложность (оценивается субъективно администратором по шкале от 1 до 4).
Для решения заданий создана специальная страница, где имеется поле ввода номера задания и при надобности id, после получения случайного задания с этим номером (если Id не указан) выводится вся информация о нем и дается поле ввода ответа, при нажатии на кнопку проверки выводится соответствующий результат правильности ответа.
### 1.4 Варианты и сборники заданий
В терминологии разрабатываемого в рамках статьи интернет-ресурса, вариант – это набор заданий, где каждое из двадцати семи может встречаться либо только 1 раз, либо не встречаться вообще. Может быть как и полным - таким же как и на КЕГЭ, либо не содержать какие-либо определенные задания. На выполнение всего варианта дается одна попытка, после завершения которой все указанные ответы проверяются одновременно, и в итоге будет показан результат, переведенный в стобалльную шкалу ЕГЭ. Сборник заданий – набор, содержащих любое количество любых задач, где правильность данного ответа на каждое задание можно проверять по отдельности бесконечное количество раз.
Создаются администратором персонально для каждого ученика. Если создается вариант, то в окне создания варианта нужно проставить галочки рядом с номером задания, которое необходимо включить. В случае создания сборника – проставляется количество заданий возле каждого номера.
### 1.5 Административная панель
Предназначена для управления всеми аспектами интернет-ресурса, позволяет регистрировать новых учеников, создавать задания, выдавать варианты и сборники заданий. Так же имеет доступ к журналу событий каждого пользователя. Запись любого события имеет вид: дата, вид отправленного материала (вариант/задание), номер задания/варианта, id, описание события. При отправке ответа в зависимости от правильности ответа выводится соответствующее пояснение, а в случае отправки варианта отображается результат и статус по каждому заданию. В случае неверного ответа выводится сам ответ, а в случае верного выводится лишь уведомление о правильном ответе.

## Реализация
### 2.1 Разработка регистрации и аутентификации
Страница авторизации была реализована при помощи HTML-форм и JavaScript и работает по следующему принципу: пользователь вводит данные в поля «логин» и «пароль», затем при помощи jQuery данные записываются в переменные и при помощи POST отправляют Ajax-запрос на PHP-скрипт, где происходит поиск аккаунта в таблице базы данных, и при нахождении аккаунта с таким логином, происходит проверка введенного пароля с правильным. В случае успешной авторизации данные об аккаунте записываются в Cookie.
 
![image](https://github.com/user-attachments/assets/a2f6525e-85e9-40ce-8aa7-b49df5a71a4e)

Создание аккаунта для ученика происходит через административную панель: в специальном всплывающем окне необходимо в текстовом формате  ввести имя, дату занятия, придумать уникальный логин и пароль.
 
![image](https://github.com/user-attachments/assets/48be6f88-723d-4dea-8fb4-b0db02e4cd1d)

После создания аккаунта, пользователь, указав корректный логин и пароль, сможет успешно авторизоваться.
Регистрация нового пользователя работает по схожему с авторизацией принципу: после ввода администратором всех данных, jQuery отправляет Ajax-запрос на PHP-скрипт, который делает запись в базе данных.
### 2.2 Реализация личного кабинета
На странице личного кабинета, куда попадает пользователь после успешной авторизации в системе, находятся:
•	Имя пользователя и дата занятия.
•	Левое боковое меню, которое содержит ссылки на главную страницу, страницу с заданиями и выход.
•	Список из всех доступных вариантов и сборников задач. Каждый из таковых имеет описание типа, дату создания и статус решения.
 
![image](https://github.com/user-attachments/assets/59cfbabd-2c3b-44d0-ad5a-ffa323f47034)

### 2.3 Разработка и создание заданий
Каждое задание будет представляться в виде записи в таблице базы данных, для его создания администратору потребуется заполнить специальную HTML-форму в админ-панели, заполнив все необходимые данные, id к каждому заданию присваивается автоматически.
После заполнения формы и нажатия кнопки «добавить» jQuery отправляет все введенные данные при помощи POST на PHP-скрипт, который создает новую запись в таблице базы данных и сохраняет файлы в соответствующих директориях.
 
![image](https://github.com/user-attachments/assets/3a8b9967-8f10-4085-9254-fc2d8af7d107)

Отдельная страница решения задания содержит следующие элементы:

•	Номер задания и его id

•	Рисунок с условием задачи

•	Необходимые для решения задачи файлы

•	Поле для ввода ответа

•	Уровень сложности

•	Кнопка проверки

После введения ответа предоставляется возможность проверки. Нажав на соответствующую кнопку, jQuery отправить на PHP-скрипт введенный ответ, после чего произойдет сравнение с правильным ответом из базы данных. В случае правильного решения задания выведется соответствующее сообщение, и затем через 3 секунды будет предложено к решению новое задание под этим же номером. В противном случае выведется сообщение об ошибке.
 
![image](https://github.com/user-attachments/assets/3065db8b-997a-47a2-b207-d75170733652)

### 2.4 Реализация вариантов и сборников заданий
Страницы, содержащие варианты и сборники заданий, представляют собой списки всех присвоенных данному варианту задач. Они создаются администраторами персонально каждому ученику. При создании вариантов открывается специальная форма (рис. 6), где необходимо выбрать те задания, которые необходимо включить в вариант.
 
![image](https://github.com/user-attachments/assets/538f8f5d-0073-449c-b143-89c29f5af5d9)

После создания варианта, у ученика в личном кабинете в списке всех материалов появляется новый вариант, при нажатии на который открывается страница с его решением.
 
![image](https://github.com/user-attachments/assets/caabe69e-fa47-4be5-a325-026adb75a691)

После завершения решения, внесения ответов и нажатия кнопки «завершить» ученику будет доступен просмотр результата и статистики решенного варианта. С серверной точки зрения проверка варианта заключается в проверке каждого задания из варианта по отдельности и в итоге формирования суммы баллов.
 
![image](https://github.com/user-attachments/assets/26aba592-2520-4ab9-9a1a-e74098463623)

Создание сборника задач состоит количества необходимых вариаций каждого задания в поле в специальной открывшейся форме админ-панели.
 
![image](https://github.com/user-attachments/assets/c94c2d6c-4d5e-44bd-8c7d-5d4ceea7347c)

Система решения сборника задач схожа с системой решения сборника, однако возле каждого задания будет кнопка проверки, и если задание решено правильно, то кнопка проверки пропадет, а поле редактирования ответа конкретно к данному задания станет недоступным к изменению. Таким образом, система дает бесконечное количество шансов на решение задач, пока оно не будет решено верно.
 
![image](https://github.com/user-attachments/assets/254724e3-b6ae-4255-b642-0ec89180d8c4)

Функционал проверки каждого задания полностью заимствован из страницы с заданиями, а формирование итоговой суммы и статуса зависит от внешней проверки при помощи  PHP-скрипта.
### 2.5 Функционал панели администрирования
Панель администрирования доступна авторизированным пользователям с особым статусом. Она позволяет добавлять варианты, задания и сборники вариантов, а так же отслеживать журнал действий каждого отдельного пользователя.
Каждый зарегистрированный ученик будет представляться в виде строки со всеми необходимыми кнопками, при нажатии на которые будут открываться соответствующие формы.

Статистика каждого пользователя представляется в виде журнала с описанием всех подробностей каждого события.
